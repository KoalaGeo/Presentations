Knowledge Extraction

Team: Rachel Heaven, Simon Burden, Bob McIntosh, Ike Nkisi-Orji, Marcus Sen, Jo Walsh

Goal

Extract stratigraphic entities and observation locations from unstructured text

Intended benefits:

Interpreted map and model products can contain links to observed data points and conceptual ideas, presenting model uncertainty and provenance

Stratigraphy of an area of interest can be quickly understood from pre-existing literature, saving experts’ time

Explicit and implicit stratigraphic relations can be captured into reusable data for modelling constraints

1. XML Semantic Markup Experiments (early 2000's)

Manual XML markup experiments in early 2000s labelled words and phrases and tagged them as members of BGS controlled vocabularies for Lithostratigraphy (BGS Lexicon) and Chronostratigraphy.

We extracted 2500 sentences from the XML and converted the mark up to a Named Entity Recognition (NER) training data format where words are classified as LEXICON or CHRONOSTRAT. NB Today various open source and commercial tools are available to create NER training datasets by annotating unstructured text.

3. Train a custom Named Entity Recognition (NER) model for strategraphic terms

BGS GitHub Repo 

							java -cp 
							edu.stanford.nlp.ie.crf.CRFClassifier - 
							prop
We used the training data to train a custom NER tagger model using the Stanford Core NLP library, which implements a CRF (conditional random field) algorithm. Pre-trained models are available that detects people, organisations and places in unseen text. Our custom trained model can also recognise lithostratigraphic and chronostratigrahic terms. NB alternatives to CoreNLP are available, we also tried python based spaCy, and Edinburgh Geoparser for detecting locations. The training data and instructions to repeat this task to build the custom NER model yourself are on https://github.com/BritishGeologicalSurvey/geo-ner-model.

4. Run text mining using custom NER model for stratigraphy and core NER locations

As a proof of concept, we used the model to tag location, lithostratigraphy and chronostratigraphy in a sample of 140 documents: half were mixed quality pdf scans of typed biostratigraphy reports that had been pre-processed using OCR; the other half were regional geological reports where the text had been converted to word then HTML with manual correction to fix OCR errors, create standardised formatting for mentions of boreholes and grid references, and create systematic document structure.

5. Stratigraphy entities matched to BGS Linked Data Vocabs

We match words/phrases tagged as LEXICON or CHRONOSTRAT to values in our controlled vocabularies (BGS Lexicon and Chronostratigraphy) and identify them using Linked Data URIs (published on BGS Linked Data pages at data.bgs.ac.uk). We use fuzzy text matching based on Levenshtein distance to account for OCR errors and to resolve the domain specific stop words (Formation, Group, Period etc).

6. Locations resolved to geometry using gazetteer

We match words/phrases tagged as LEXICON or CHRONOSTRAT to values in our controlled vocabularies (BGS Lexicon and Chronostratigraphy) and identify them using Linked Data URIs (published on BGS Linked Data pages at data.bgs.ac.uk). We use fuzzy text matching based on Levenshtein distance to account for OCR errors and to resolve the domain specific stop words (Formation, Group, Period etc).

7. Database of mined data with API

We store the mined and entity matched data in a database (containing: document id, token start and end position, entity type, vocabulary match URI or geometry) and wrote a simple API to query it.

8. Marked up text linked to map

We built a user interface that queries the API and details the lithostratigraphic and chronostratic terms found in each document, plots locations on a map overlaying the digital geological map, and links each vocabulary term or map to the source text snippets. Each matched entity can also launch a link to further information provided by the gazetteer or Linked Data vocabulary service.

9. API supports frequency and corrolation analysis

The API querying the matched entities database supports term frequency plots, synthetic “generalised vertical section” and simple correlation analysis based on character distance in text. Further work can be done here to analyse the text for the nature of the correlations, hopefully extracting explicit descriptions of stratigraphic relations such as “overlies”, “deposited unconformably on” etc, and relating stratigraphies to the locations they were observed at.
